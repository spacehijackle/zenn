前回は「体力の限界！気力もなくなり…」と、あの有名なフレーズよろしくエネルギー切れで記事を打ち切ってしまいましたが、今回もダマシダマシでやっていきます（　＾ω＾）・・・

https://zenn.dev/spacehijackle/articles/df9c9b8cf8fbdb

前回はチェック・ボックスとラジオ・ボタンのコンポーネント自体の説明が多かったのですが、今回はその応用、というかサンプル・プログラムでいくつかの **“宿題”** を確認していきます。

> しかし、実際にはラジオ・ボタンを横に並べたい時もあるでしょう。もしくは、ラジオ・ボタンの間に他のコンポーネントを挟み込みたいこともあるでしょう。つまり、フロントであるRadioButtonGroupは、どのようにラジオ・ボタンが配置されるか？については呼び出し元に委ねるしかありません。だって、どのように配置したいのか分からないのですから。。。

前回、`RadioButtonGroup`は柔軟なコンポーネントの配置を可能にする設計にしたよ、って旨をお伝えしましたが、ここでその証明です。

![](/images/articles/ed4353c15520ab/guitar01.jpg)

前回のサンプル・プログラムでは複数のラジオ・ボタンの間に、その他のコンポーネントは存在していませんでしたが、今回はラジオ・ボタン間に`Text`, `Spacer`を挿入してみました。ソースは以下に表示しますが、後ほど、改めて詳しく見ていきます。

https://github.com/spacehijackle/SwingUI_07/blob/main/app/src/main/java/swingui_07/guitar/GuitarCommentary.java

ちなみに`Guitar`クラスは以下です。

https://github.com/spacehijackle/SwingUI_07/blob/main/app/src/main/java/swingui_07/guitar/Guitar.java

外部からのインスタンス生成を制限し、内部で定数としていくつかのギター種類を定義しています。

改めて画面の記述を見ていきます。

```Java:GuitarCommentary.java
public class GuitarCommentary
{
    // ギターのタイプの選択値
    private final UIValue<Guitar> selected = UIValue.of(Guitar.ELECTRIC);

    // ギタータイプの解説テキストのフォントカラー更新クラス
    private CommentColorUpdater fgUpdater = new CommentColorUpdater();

    void build()
    {
        Frame.of
        (
            "ギターの種類",

            RadioButtonGroup.of
            (
                selected,
                VStack.of
                (
                    UIAlignmentX.Leading,  // 左寄せ

                    ///
                    /// エレキ・ギター
                    /// 
                    RadioButton.of(Guitar.ELECTRIC),

                    Text.of(Guitar.ELECTRIC.comment)
                        .foreground(fgUpdater.electric)
                        .padding(Horizontal.of(16)),

                    Spacer.of(Height.of(8)),

                    ///
                    /// アコースティック・ギター
                    /// 
                    RadioButton.of(Guitar.ACOUSTIC),

                    Text.of(Guitar.ACOUSTIC.comment)
                        .foreground(fgUpdater.acoustic)
                        .padding(Horizontal.of(16)),

                    Spacer.of(Height.of(8)),

                    /********** ＜中略＞ **********/
                )
            )
            .padding(24)
            .onCheckChanged((guitar) -> fgUpdater.update())  // 選択値の変更の度に解説テキストのフォント色更新
        );
    }
}
```

`RadioButtonGroup`の第一引数はグループ内での選択値を表す変数です。初期状態でエレキ・ギターを指定しているので、ラジオ・ボタンもそれと呼応してエレキ・ギターが選択されています。

そして第二引数ですが、`VStack`で囲った中に、`RadioButton`や`Text`,`Spacer`を並べていますが、これによりラジオ・ボタン以外のコンポーネントを加え、グルーピングで単一選択を可能にしつつも、柔軟なコンポーネント配置が可能になっていることが分かるでしょう。まぁ`VStack`で囲うのは前回のサンプル・プログラムにもあったので、特別なこともないのですが、本当にラジオ・ボタン以外のコンポーネントも表示できるよ、っていうのをお伝えしたかったんです。。。

そして`RadioButtonGroup`の`.onCheckChanged()`で選択されたギターを感知できるようにしていますね。ここでは選択したギター以外の解説テキストをグレー表示にする“演出”を施しています。各`Text`に対し、前景色を`UIValue`を利用して更新することで可能になっています。色の指定自体は`CommentColorUpdater`で行っています。なお`CommentColorUpdater`は`GuitarCommentary`の内部クラスとなっていて、変数`selected`を共有しています。

```Java:GuitarCommentary.java
    class CommentColorUpdater
    {
        // 選択状態のフォントカラー
        private final Color SELECTED = Color.darkGray;

        // 非選択状態のフォントカラー
        private final Color UNSELECTED = Color.gray;

        //
        // ギタータイプ別の解説テキストのフォントカラー
        //
        private UIValue<Color> electric = UIValue.of(UNSELECTED);
        private UIValue<Color> acoustic = UIValue.of(UNSELECTED);
        private UIValue<Color> classical = UIValue.of(UNSELECTED);
        private UIValue<Color> airy = UIValue.of(UNSELECTED);

        /**
         * 各ギタータイプ解説のフォントカラーを選択状態に応じて更新する。
         */
        void update()
        {
            electric.set(Guitar.ELECTRIC.equals(selected.get()) ? SELECTED : UNSELECTED);
            acoustic.set(Guitar.ACOUSTIC.equals(selected.get()) ? SELECTED : UNSELECTED);
            classical.set(Guitar.CLASSICAL.equals(selected.get()) ? SELECTED : UNSELECTED);
            airy.set(Guitar.AIR.equals(selected.get()) ? SELECTED : UNSELECTED);
        }
    }
```

`update()`で選択中のギターと各ギターを比較し、同じかどうかで色を変えています。


## 本題はこれからだ、お嬢ちゃん😎 ～forEach拡張～

選択肢が固定ならば良いのですが、選択肢が何かのリストデータを基に作成する場合だってあるでしょう。そうなると、リストのデータを一つずつ取り出し、それを元にしたコンポーネント生成を可能にしなければなりません。

`SwiftUI`では、同様の処理を`forEach`で対応するのですが…

```Swift
struct ContentView: View
{
    let guitars = ["Electric Guitar", "Accostic Guitar", "Classical Guitar"]

    var body: some View
    {
        List
        {
            ForEach(guitars, id: \.self) { guitar in
                Text(guitar)
            }
        }
    }
}
```

ただ`Java`でこの記述方法を使うこと自体できないので、別の方法を考えなければなりません。と、言うことで考えました。`VStack`に`forEach`メソッドを設けてみました。

まず、以下のソースをご覧ください。

```Java:GuitarCommentary2.java
public class GuitarCommentary2
{
    // ギターのタイプ一覧
    private final List<Guitar> guitars = Arrays.asList
    (
        Guitar.ELECTRIC, Guitar.ACOUSTIC, Guitar.CLASSICAL, Guitar.AIR
    );

    // ギターのタイプ選択値
    private final UIValue<Guitar> selected = UIValue.of(Guitar.ELECTRIC);

    // ギタータイプの解説テキストのフォントカラー更新クラス
    private CommentColorUpdater fgUpdater = new CommentColorUpdater();

    public GuitarCommentary2()
    {
        fgUpdater.update();
        build();
    }

    private void build()
    {
        Frame.of
        (
            "ギターの種類 - VStack#forEach(...)",

            RadioButtonGroup.of
            (
                selected,
                VStack.forEach
                (
                    UIAlignmentX.Leading,
                    guitars,  // ギターのタイプ一覧
                    (idx, guitar) ->
                    {
                        // ギターのタイプ毎のラジオ・ボタン選択とその説明テキストを追加
                        return VStack.of
                        (
                            UIAlignmentX.Leading,

                            RadioButton.of(guitar, (g) -> String.format("%d. %s", (idx + 1), g)),

                            Text.of(guitar.comment)
                                .foreground(fgUpdater.colorMap.get(guitar))
                                .padding(Horizontal.of(16)),

                            // 末尾以外、余白を追加
                            (idx != (guitars.size() - 1) ? Spacer.of(Height.of(8)) : Spacer.empty())
                        );
                    }
                )
            )
            .padding(24)
            .onCheckChanged((guitar) -> fgUpdater.update())  // 選択値の変更の度に解説テキストのフォント色更新
        );
    }
}
```

`guitars`というリストにギターの一覧を定義し、これを`VStack#forEach()`に渡しています。このメソッドではリストから一つずつデータ（ここではギターの種類データ）を取り出し、これを呼び出し元へ渡します。渡された呼び出し元は、このデータを基にコンポーネント配置した結果をリターンします。ここではラジオ・ボタンと解説テキストのテキスト、最後に余白ですね。リターンを受け取ると、`VStack`のメソッドなので、これを縦に配置。これでリストデータをグルグル回しながらのコンポーネント配置が可能になります。

![](/images/articles/ed4353c15520ab/guitar02.jpg)

リストのデータを渡すときに、リスト位置のインデックスも渡しているので、これを基にラジオ・ボタンのテキストに番号表示をしたり、最後の余白を追加しなかったり、という操作が可能になっています。ちなみに`Spacer.empty()`はサイズゼロの余白スペーサですが、実際はこのゼロスペーサを縦に並べてる、のではなく、サイズゼロのコンポーネントは縦に並べる際に除外しています。なぜなら、`VStack`はコンポーネントを縦に配置する際、コンポーネント間を予め設定されたスペースを確保した上で並べるため、サイズゼロであっても、設定されたスペース分が確保されてしまいます。これは設定スペースが大きければ気になるでしょう。サイズゼロのコンポーネントなのになぜかスペースが追加されてるぞ！と思われないための対応策です。ちょっと細かい話ですが、一応触れておきますね。

```Java:VStack.java
    public static <E, T extends JComponent> PanelWT forEach
    (
        UIAlignmentX alignmentX, Spacing spacing, List<E> list, BiFunction<Integer, E, T> generator
    )
    {
        List<T> components = new ArrayList<>();
        for(int i=0; i<list.size(); i++)
        {
            T child = generator.apply(i, list.get(i));
            if(child != null)
            {
                components.add(child);
            }
        }
        return of(alignmentX, spacing, components.toArray(new JComponent[components.size()]));
    }
```

`VStack#forEach(...)`を抜き出してみました。受け取ったリストを`for`文で回し、`BiFunction`である`generator`にデータを渡しています。`BiFunction`は`Java`標準で用意された関数型インターフェースで、二つの引数を取り、その結果を返す仕組みです。ここでは二つの引数とは、①インデックス、②リストのデータですね。`BiFunction#apply()`によって実行されます。このメソッドによって得られたコンポーネント配置された返却値が、次々に追加され、最終的に縦に並べられる、といった仕組みです。これと同様の仕組みは`HStack`にも追加しておきました。
